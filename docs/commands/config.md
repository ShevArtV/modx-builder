# config — Глобальные настройки

Интерактивно настраивает глобальный конфиг Package Builder. Эти настройки используются как значения по умолчанию для всех проектов.

## Использование

```bash
modxapp config
```

## Что настраивается

| Параметр | Описание | По умолчанию |
|----------|----------|-------------|
| `author` | Имя автора | `Your Name` |
| `email` | Email автора | `your-email@example.com` |
| `gitlogin` | Git username | `your-username` |
| `phpVersion` | Минимальная версия PHP | `8.1` |
| `repository` | URL репозитория | — |
| `template` | Путь к шаблонам по умолчанию | встроенные |
| `generateElements` | Генерировать файлы элементов | `true` |
| `phpCsFixer` | Добавлять PHP CS Fixer | `false` |
| `eslint` | Добавлять ESLint | `false` |

## Где хранится

Если вы не выполняли `modxapp config`, используются значения по умолчанию (автор — `Your Name`, email — `your-email@example.com` и т.д.). При первом запуске `modxapp config` создаётся ваш персональный конфиг `~/.modxapp/config.json`, который затем используется как основа для всех проектов — вам не придётся вводить одни и те же данные каждый раз.

Конфиг хранится в домашней директории пользователя и не затирается при обновлении Package Builder.

## Связь с локальным конфигом

При выполнении `modxapp init` или первом `modxapp create` значения из глобального конфига копируются в локальный `modxapp.json`. После этого проект использует только локальный конфиг.
